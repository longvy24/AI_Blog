Why The Retry Spiral Doesn't Work

Same Prompt Class, Same Bug

Watch what most people do when ChatGPT returns a bad answer: "Make it better." "More detail." "Different angle." "Try again." Five rewordings of the same vague instruction, each producing a slightly different shade of the same generic output. Twelve minutes burned. Same bug.

The retry spiral feels like progress because you're doing something. You're typing. You're hitting send. The output is changing. But the prompt is the same shape — you're describing the same problem with different adjectives and the model is doing the same thing it always does with vague input. You can't fix the output without changing the input class.

The fix is to stop and diagnose. Read the bad output and ask: why is it bad? Not "what's wrong with the model" — what's missing from your prompt that the model needed to do better? The answer is almost always one of three things.

★ Retry blind
~12m
★ 5 rewordings · same bug
★ Triage first
~30s
★ 1 diagnosis · 1 re-prompt

"Don't retry — triage. Five blind rewordings beats one diagnosed fix. Same model. Same prompt class. Same bug. Every time."

The Triage Chart

3 Symptoms. 3 Fixes.

Three failure modes cover the vast majority of bad AI replies. Memorize them in the order they're most likely to bite you. When the output comes back wrong, read it, match it to one of these three, and apply the one matching fix. Don't apply two. Don't reword. Diagnose, patch, ship.

★ 01 · Symptom
Vague Output
"helpful but generic." Sounds like Wikipedia. No concrete numbers, no named entities, no specific examples — the model is hedging because you let it.
→
✓ Fix · Add Specifics
Numbers · Names · Examples · Domain
Replace adjectives with quantities. Replace "audience" with a named persona. Replace "improve it" with "include this stat." Specifics force specifics.
★ 02 · Symptom
Wrong Format
Wall of text when you wanted a table. Five bullets when you wanted JSON. 800 words when you wanted 80. The shape is wrong even if the content is right.
→
✓ Fix · Name The Shape
Table · Bullets · JSON · Markdown · Word Count
Be exact about the container. "2×4 markdown table." "Max 90 words." "Strict JSON, no prose." The model only delivers the shape you name.
★ 03 · Symptom
Missed Context
The model didn't know about your stack, your brand voice, your data, your codebase, your previous decisions. It guessed — generically. Confidently. Wrong.
→
✓ Fix · Paste The Source
Docs · Screenshot · Prior Output · Examples
Stop describing context — paste it. Drop the doc, the screenshot, the previous reply, the brand guide. The model can read what you can read.
Three Real Triages

Bad → Triaged → Shipped

Three real prompts I've triaged in the last week. Each follows the same shape — bad input, one diagnosis, one fix applied. The output speaks for itself.

★ Case 01 · Vague output → + Specifics
"Tips for a strong opener"
Bad output reads like a marketing blog. Generic advice. No way to act on it. Diagnose: vague. Fix: add specifics.

"Compare these vendors"

Why It Works

You're Changing The Input Class

The retry spiral fails because every rewording is the same kind of prompt. You're varying the surface — different verbs, different adjectives — but the underlying structure is identical. Vague-in, vague-out. Format-blind in, format-blind out. The model is doing its job; you're handing it the same job five times.

Triage works because you're changing the input class. Add specifics shifts the prompt from "describe a topic" to "produce content with these exact ingredients." Name the shape shifts it from "answer a question" to "fill this container." Paste the source shifts it from "guess at my context" to "operate on this artifact." Each fix is a different category of instruction — and that's why the output category changes too.

Build The Habit

The 4-Step Protocol

  • Read the bad output. Don't react. Just read it. ~5 seconds. What is wrong about it — vague, wrong shape, missing info? (Spoiler: it's one of those three.)
  • Diagnose. Match the failure to one of the three symptoms. ~5 seconds. Don't try to find two — pick the dominant one.
  • Patch. Apply the matching fix and only the matching fix. ~10 seconds. Add specifics, or name the shape, or paste the source. Resist the urge to triple-fix.
  • Ship. One re-prompt. ~10 seconds. Don't reword the whole prompt — just add the patch on top of the original. Hit send.
  • Audit your last week: how many times did you retry the same prompt 3+ times? Each one was a triage opportunity. Bookmark this page, paste the chart somewhere, and run the protocol on the next bad reply.
Common Mistake

Don't Apply All Three Fixes

The most common triage failure is shotgunning. You read the bad output, panic, and add specifics AND name the shape AND paste the source all in one re-prompt. Sometimes it works — but if it doesn't, you have no idea which fix did or didn't apply. You can't learn from a multi-fix re-prompt. One symptom, one fix. If you're wrong about the diagnosis, the next re-prompt is still under 30 seconds.

The other mistake is rewording the whole prompt from scratch when you only need to add one line. The original prompt isn't the problem — the missing piece is. Keep the original, append the fix. The model handles "old prompt + new constraint" better than "totally new prompt that's almost the same."

// The One Thing To Remember

Stop retrying. Start triaging. Three symptoms cover almost every bad AI reply — vague, wrong format, missed context. Each has exactly one fix — add specifics, name the shape, or paste the source. One diagnosis, one re-prompt, 30 seconds. Beats the 12-minute spiral every single time, and you'll learn to spot the symptom before you even hit send next time.
