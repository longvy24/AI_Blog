Two Script Errors To Fix

This video's script contains two factual errors — one product name hallucination, one pricing claim that doesn't hold up. Both are documented below before the tool breakdowns.

// Correction — Script Named a Non-Existent Product

What the script said: "Number two — Perplexity Computer. It's now a full multi-agent browser assistant. It can open your calendar, look up meetings, write briefing docs — all from one prompt. It just became available to all Pro subscribers."

The product name error: "Perplexity Computer" does not exist. The script — generated by Claude from the RFTC model — hallucinated the product name. Keyvello correctly defaulted to Perplexity AI in the video, which is the real product.

What the script actually described — turns out — is real: The feature described (calendar access, looking up meetings, writing briefing docs from one prompt) is a documented capability of Perplexity AI when its Gmail and Google Calendar connector is enabled. The script had the right use case, the wrong product name. The Tool #2 card below covers what Perplexity AI actually does, sourced from Perplexity's own documentation.

What this blog post got wrong initially: The first draft researched "Perplexity Computer" as if it were real and wrote a full breakdown for a fabricated product. That content has been removed and replaced with sourced information about the actual product.

// Correction 2 — n8n Is Not Free

What the script said: "Think of it as Zapier, but free and 10 times more powerful."

What's accurate: n8n's cloud plans start at $20/month (Starter, billed annually, 2,500 executions) and $50/month (Pro). There is no free cloud tier. The Community Edition is free to self-host, but that requires downloading, installing, and maintaining it on your own server. For a general audience watching a short-form video, "free" implies you can start using it right now at no cost. You can't — not on the cloud version, and not without technical setup on self-host.

"Two script errors in one video: a product name that doesn't exist, and a tool described as free that starts at $20/month. The use cases were right. The facts needed work."

The Three Tools

What They Actually Do

The video runs as a Top-N countdown — n8n first (labeled "Tool 3" in the count, shown at 8–25s), then Perplexity AI (Tool 2, 25–42s), then Claude with MCP (Tool 1, "number one," 42–55s). Here's what each actually is, with the script error documented where it applies.

3
// Tool #3 in countdown · Revealed first · 8–25s
n8n

What the script says: "First — n8n. This is an open-source automation tool that connects over 400 apps together and runs them on autopilot. Think of it as Zapier, but free and 10 times more powerful. You set up an agent once, and it handles emails, data entry, CRM updates — without touching a single button."

What's accurate: n8n is a source-available, self-hostable workflow automation platform. The "400+ apps" figure is correct per n8n's own features page. The "free" claim in the script is not accurate as stated. n8n's cloud plans start at $20/month (Starter, billed annually, 2,500 executions) and $50/month (Pro, billed annually, 10,000 executions). There is no free cloud tier. The Community Edition is free to self-host — meaning you download and run it on your own server — but that requires technical setup and ongoing infrastructure. For someone watching a short-form video, "free" implies they can use it for free right now. They can't, unless they're comfortable self-hosting.

The "10x more powerful than Zapier" claim is unverifiable as stated. The real structural advantage is n8n's execution-based billing model: one complete workflow run counts as one execution regardless of how many nodes it contains. Zapier charges per step. A 20-node n8n workflow costs the same as a 2-node one — on Zapier, it costs 10x more. For complex automations, this makes n8n dramatically cheaper at scale.

Beyond basic integrations, n8n supports AI agent workflows natively — you can connect LLMs like OpenAI or Gemini directly into workflows, build multi-step reasoning chains, and create agentic systems that classify data, make decisions, and trigger actions autonomously. The platform also supports custom JavaScript and Python in code nodes, giving developers full control without losing the visual builder.

Bottom line: The script's "open-source Zapier alternative with 400+ integrations" framing is accurate. "Free" is not — cloud plans start at $20/month. Self-hosting is free to download but requires your own infrastructure and setup. "10x more powerful" is unverifiable marketing language. Two out of three claims in the script's n8n description need correction.

  • Automate email triage: route, label, and forward based on content — without touching a button
  • Sync data between a CRM, database, and Google Sheets on a schedule
  • Build an AI agent that classifies support tickets and triggers the right follow-up workflow
  • Expose a complex automation as an API endpoint using webhook nodes

// Tool #2 in countdown · 25–42s
Perplexity AI

What the script said: "Number two — Perplexity Computer. It's now a full multi-agent browser assistant. It can open your calendar, look up meetings, write briefing docs — all from one prompt. It just became available to all Pro subscribers."

Product name error: "Perplexity Computer" doesn't exist — see correction above. The tool is Perplexity AI.

What Perplexity AI actually does — and the script's description is closer to right than I first said: Perplexity AI has a native Gmail and Google Calendar connector. Once enabled, you can query your inbox and calendar directly from Perplexity — it searches across your email, calendar events, connected apps, and the web in a single interface. You can ask it to summarize emails, see who you're meeting with this week, and generate contact bios — all from one prompt. Per Perplexity's own documentation, a query like "Prepare me for my next meeting with [Company]" pulls past email threads with your contacts, meeting history from your calendar, contact background, company overview, and recent news — returning a structured briefing document with cited sources.

So the use case the script describes — calendar, meetings, briefing docs — is a real documented feature of Perplexity AI. The script had the right idea, the wrong product name.

Availability: The Gmail and Google Calendar connector works within Perplexity AI. The Comet browser integration (which adds the ability to create calendar events and send emails directly) requires Comet. Free plan includes limited daily Pro searches; Pro is $20/month. The connector itself is available at the account level once enabled in Settings → Connectors.

  • Summarize the emails I received yesterday and prioritize the ones that need my attention
  • Who am I meeting with this week? Write bios for each person
  • Prepare me for my next meeting with [Company] — pull past conversations, their recent news, and suggest three discovery questions

Honest Disclosure

The "4 Hours Saved" Claim

The script opens with: "Number 3 alone saved me 4 hours this week." This line was written by Claude as part of the script generation. I did not personally measure this. It's a hook device, not personal testimony — and I should have flagged it as such before the video published.

This is a pattern worth addressing directly. The RFTC model instructs Claude to optimize for short-form engagement, and hooks like "saved me 4 hours" are exactly what that produces. They work. They're also unverifiable unless I actually track it, which I haven't. Starting from post 04, any time-savings or ROI claim in a script either gets sourced from a real test, or gets labeled explicitly as a scripted hook — not stated as personal fact.

The Numbers

Why 213 Views Couldn't Cross Over

This video is the second-best performer in the series so far — 213 YouTube views as of May 13, behind only video 01 (1,002 views). On TikTok, 94 viewers, drop-off at 0:02. On Instagram, 23 views and zero interactions on any metric: likes, comments, saves, shares all at zero.

Three videos, three TikTok cliff drops at 0:01 or 0:02. The retention curve is the same every time — immediate drop, then a flat line of near-zero viewers for the rest of the video. This isn't a content quality problem. It's a hook format problem. The current hook — text on screen while the voiceover reads that same text — isn't creating any visual tension in frame zero. TikTok viewers decide in under two seconds, and nothing in the opening frame is stopping the scroll before the words arrive.

The YouTube story is different. 213 views without a meaningful subscriber base suggests the algorithm is distributing the video to non-subscribers through Shorts browse and suggested video. The same content that fails at 0:02 on TikTok can still compound on YouTube, where the discovery mechanism and watch-time calculation work differently.

// Three-Video Pattern

YouTube: 1,002 → 266 → 213. Declining but still showing organic reach. TikTok: 0:01, 0:02, 0:02 drop-off across all three videos. The hook structure isn't working on that platform. Instagram: consistently 23–131 views with near-zero engagement across all posts. The fix for TikTok is not in the script — it's in what the first frame looks like before any voiceover begins.

TikTok's viewer data: 100% non-followers, 98% new viewers. Every person who saw this video found it through discovery — not because they subscribed. That's expected at this stage. But it means each video needs to earn its own audience from scratch, and two seconds is all it gets to do that on TikTok.
