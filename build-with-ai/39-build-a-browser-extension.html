The Extension I Wanted

A Browser Extension Was Always "Ask a Developer"

A browser extension is the classic "you'd need a developer" project. It sounds technical — a manifest, permissions, content scripts, the Chrome APIs. So the little tools you actually wish existed while you're browsing? You just live without them.

The one I always wanted: I read a lot of long, dense articles for this channel, and I wanted to highlight a paragraph and instantly get the gist — without leaving the page. So I described it to Claude. The result is Gist: select any text on any page, and a popup summarizes it, explains it simpler, or rewrites it. Powered by Claude. It wrote every line.

// The Pitch I Gave Claude
Build a Chrome extension. When I highlight text on any page, show a small popup that can summarize it, explain it simpler, or rewrite it — using the Claude API. Give me the manifest, the popup UI, and the content script, ready to load unpacked.

That was the seed. Claude wrote the whole extension — and I shaped it by using it. The manifest, the popup with its three modes, the content script that grabs your selection, the call to Claude. I wrote zero lines of code. I loaded it into Chrome unpacked in one click, and it just worked.

"Now I highlight anything — a dense paragraph, a contract, a recipe — and Claude hands me the gist in seconds. On any page, without leaving it."

Do It Yourself

Build the Tool You Keep Wishing For

You've had the thought a hundred times: "I wish my browser just did X." That X is an extension, and it's a description away. Here's the loop:

  • Name the one job. "Summarize what I highlight." "Strip the paywall clutter." "Save this to my notes." One extension, one job — that's the whole trick.
  • Describe the trigger. Tell Claude how it starts: on highlight, on a toolbar click, on page load. That single sentence decides the whole shape of the thing.
  • Ask for it ready to load. Have Claude output the manifest, the popup, and the script together, "ready to load unpacked." Then drop the folder into Chrome's extensions page in developer mode.
  • Refine by using it. "Make the popup smaller." "Add a copy button." "Explain it like I'm five." You use it on real pages; Claude tunes it.
Why It Works

The Wall Was Never the Idea

Everyone who browses the web has a running list of tiny "why doesn't this just…" wishes. The idea was never the problem — the wall was the Chrome extension plumbing: the manifest format, the permissions model, the message-passing between a content script and a popup. It's fiddly, it's specific, and it's exactly the kind of well-documented pattern AI has cold.

So the skill that matters isn't knowing the Chrome API — it's noticing the wish and describing it clearly. You don't need to be a developer to build a browser extension. You need to know what you want your browser to do.

The Honest Part

What AI Builds and What You Do

No hype — building a working extension this way is genuinely a one-afternoon thing. But an AI-powered one has one real catch. Here's the honest split.

✓ AI handles
  • The whole extension — manifest, popup, content script.
  • Turning a description into a folder you load in one click.
  • The Chrome-API plumbing you'd otherwise have to learn.
  • Tweaks and new modes, on request.
! Still on you
  • The API key — an AI extension calls Claude, so you add your own key. Never paste it into a random site; keep it in the extension.
  • Knowing the one job you actually want it to do.
  • Publishing (the Chrome Web Store has its own review) — though running it unpacked, just for you, needs none of that.

The honest promise: AI removes the "you'd need a developer" wall completely — the code, the manifest, the plumbing. The one thing it can't do is hand you an API key; a tool powered by Claude needs your key to talk to Claude. Add it once, keep it in the extension, and you've got a private tool nobody had to build for you.
