// THE PRODUCTION AUDIT
The Setup

Ten Videos. Thirty Renders.

Vol 2 is a single connected series — ten lessons on prompt engineering, each one screenshot-able, each one paired with a full blog post, each one cycled across a fixed three-color palette (lime, blue, red, repeating). Topics run from context feeding through system prompts, temperature, formatting control, tree of thought, persona depth, prompt templates, multimodal prompting, and finally prompt triage as the closer.

The biggest production shift from Vol 1 is the variant policy: every brief gets three independent video builds — A (the original aesthetic, the foundation of the brand), B (an app-chrome reframe — vision.chat, persona.app, triage.app, etc.), and C (an illustrated diagram reframe with a paginated lesson-page worksheet feel). Same VO, same brief, three different visual stories. Each one feeds a different algorithm preference.

What Changed

From Keyvello to HyperFrames + GSAP

The first 10 videos were end-to-end Keyvello — AI script, AI voiceover, AI visuals, AI captions, AI assembly. It shipped fast and crashed visibly. The entire prompting series — Vol 1 and Vol 2 alike — swapped Keyvello for a deterministic stack: HyperFrames HTML compositions, GSAP timelines, Kokoro TTS, Tailwind v4 runtime styling, hand-written content. Every frame is reproducible. Every voiceover is on disk before render. Nothing hallucinates a product name.

★ The First 10 · Keyvello era
  • Pipeline: Keyvello end-to-end (script + VO + visuals + assembly)
  • Cadence: 1 video per topic · ship-and-pray
  • Review loop: none baked in
  • Hallucinations: 30+ documented errors
  • Examples: "Perplexity Computer" · "$29 Perplexity Pro" · "Qaurterly Sals Report"
  • Audio: ElevenLabs (embedded) — mono defects, platform bleed
  • Result: 4,407 views, 2 breakouts, 8 craters
★ Prompting series · HyperFrames era
  • Pipeline: HyperFrames + GSAP + Tailwind + Kokoro + Whisper
  • Cadence: 3 variants per topic (A original / B app-chrome / C illustrated)
  • Review loop: lint + validate + inspect on every change
  • Hallucinations: 0 — content is hand-written, VO is deterministic
  • Examples: 4 fully-rendered templates per video, all hand-verified
  • Audio: Kokoro am_adam, transcribed with Whisper for QA
  • Result: Vol 1 shipped — 3 followers, hook thesis proven. Vol 2 now live — 69% of recent YouTube views, the channel's #1 video, first subscriber.
The Variant Pattern

Three Stories. One Brief.

Here is the production model that defines Vol 2 — and it's the single biggest bet of the volume: the script never changes, but the video changes completely. Each lesson gets one Kokoro voiceover, written and recorded once. That one script is then built into three entirely different videos — not the same video recolored, but three distinct visual worlds with their own layouts, motion, on-screen language, and titles. Same words in the viewer's ears; a completely different thing on the screen.

And they don't all post to the same place. Each platform gets its own version. The dashboard build, the mock-app build, and the illustrated-worksheet build are distributed one per platform — so TikTok, Instagram, and YouTube are each running a visually different video of the identical script, each matched to what that platform's algorithm rewards. The feed can't flag them as duplicates, and every audience gets the aesthetic it responds to. The proof is already in the data below: the same System Prompts lesson shows up as "ChatGPT Keeps Forgetting What You Said" on YouTube and the app-chrome "Configure ChatGPT Once" on Instagram — one script, two completely different videos, two platforms. Vol 1 proved the platforms want different things; this is the answer in one line: one script, three videos, three platforms.

The Stack

What Replaced Keyvello

Six tools doing what one tool used to do badly. Each one is open source or free-tier. Each one is deterministic — same input, same output. The whole stack lives on disk, runs locally, and can render in parallel.

One honest caveat: the six tools below are free, but the loop that drives them isn't. Every video in Vol 2 was authored inside Claude Code on a Claude Max subscription — scene layout, GSAP timing, lint chasing, blog drafts, all of it. Realistic throughput is roughly one video build per working session before the token budget catches up, not a render farm churning out shorts in an afternoon. The render itself is fast; the human-plus-Claude pair authoring the composition is the rate-limited step.

★ Composition
HyperFrames
HTML + CSS + JS compositions that render to MP4. npx hyperframes for init, lint, inspect, preview, render. Apache 2.0, runs Chrome under the hood, 1080×1920 vertical at 30 fps.
★ Animation
GSAP
Single paused timeline registered on window.__timelines, seeked by HyperFrames during render. Every fade, every stagger, every scale is in one place — easy to retime when the VO shifts.
★ Voiceover
Kokoro · am_adam
Local TTS, 5 VO files per video at speed 1.0, ~19–22s total speech per render. Same voice across all 30 renders. Zero ElevenLabs API calls.
★ Transcription
Whisper · small.en
Used to audit VO text against intent — caught at least two miswordings before render. Also used for word-level timing if captions are ever added.
★ Styling
Tailwind v4 runtime
Browser-runtime Tailwind, no build step. Custom CSS variables per scheme (lime / blue / red), shared across all three variants. Same composition, different palette.
★ Stock media
Pexels MCP
Added late in Vol 2 — direct Pexels search and download from inside Claude Code via a free API key. 9 tools exposed, 25,000 requests/month. Used for backdrop B-roll going forward.
What Worked

Production Wins

  • The 5-scene skeleton. Every video locks the same structure: Hook (0–5.5s), Anatomy (5.5–13s), 3 Cases (13–20s), Principle (20–26s), CTA (25.5–37s). Once that frame is wired, the VO timing, scene transitions, and pacing memory rules (~2s VO gaps, 3.5s power2.in BGM fade) all just slot in. The first video took a week. The tenth took an afternoon.
  • The color-scheme rotation as a brand signal. Lime → Blue → Red, cycling. Watchers who binge three in a row see distinct palettes but consistent typography, header style, and pacing. Same brand, different mood per video.
  • The "fill the white space" iteration rule. Every scene gets reviewed for empty vertical space. The fix is never to shrink the existing content — it's to add value (AI output proof blocks, before/after live previews, starter-line reference grids, stats bands). By V18 this was the default. By V20 every empty-space audit was solved by adding a new content block.
  • The A/B/C variant trio. Three independent renders per brief. The aesthetics are recognizable on sight: A = dashboard, B = mock app, C = lesson page. When the algorithm sorts these into different audiences, we'll know which aesthetic each platform's audience prefers — and we'll have data to dial it.
  • Output blocks as the "show, don't tell" payoff. Every video that demonstrates a fix or template now closes the loop with a rendered AI output sample — italic monospace, accent-colored border-left, "↪ AI Output" label. It turns abstract advice into visual proof. Lifted directly from V19's multimodal pattern and back-ported to V18 and V20.
  • Memory files persist preferences across sessions. Audio pacing defaults, variation policy, Pexels download dir, video output rollover rule — all live in ~/.claude/projects/.../memory/. Future sessions load them automatically. No re-asking, no drift.
What Was Hard

The Stuff That Didn't Just Happen

  • Iteration cycles on C variants. V17C, V18C, and V20C each went through 3–4 rebuilds. V18C in particular was rebuilt as a paper-worksheet aesthetic, then ported back to dark digital after feedback that "it didn't match the other videos." The lesson: lock the aesthetic family early; iterate on content, not on the whole vibe.
  • Layout overflows under GSAP transforms. Multiple times, the HyperFrames inspector flagged elements that were "in spec" at rest but overflowed during the GSAP fromTo's initial state (e.g., x: -40 initial translate pushing a card outside its grid track). Fix: switch to y-based reveals where possible, or use opacity + scale as initial states instead of x.
  • Contrast warnings on lime/blue stamps. The HyperFrames validator flags dark-text-on-bright-accent as a contrast failure on every render. That's the design intent (★ SAVE button, ⏱ 10s FIX stamp, ★ HOMEWORK badge) — but the warning count was 50+ per video by V20. Accepted as known false positives; documented in the per-video metadata so future audits don't re-flag them.
  • VO drift between transcript and brief. Twice in Vol 2 the rendered VO didn't match the brief I'd written (e.g., V18B initially showed 8 templates but the VO said "four template categories"). Whisper transcription caught these — the discipline now is to transcribe every VO and visually diff against the brief before locking the composition.
  • White-space audits. User feedback on V17, V18, V19, V20 all hit the same beat — "card 2/card 3 has too much empty space." The fix is the same content-add pattern every time (proof block, stats band, starter lines). The miss is not anticipating it. Vol 3 design rule: every scene must fill its full vertical budget at draft, not at iteration.
  • The PDF cheatsheet URL change. Late in Vol 2 the user changed the public blog host from Netlify to a GitHub Pages URL. The PDF had to be patched in-place — overlay rectangles drawn over the old URL text, new URL drawn on top, link annotations injected. Worked, but it's the kind of asset patch that's easier to avoid by parameterizing the URL at build time.

The Playbook

Vol 3 — Locked-In Changes

Some of this is pre-emptive (responding to predictable failure modes from Vol 2), some of it is structural (decisions the user has already made for the next 10). All of it is locked before Vol 3 work begins.

  • New output directory rule. Vol 3 renders land at AI_BlogSite/<PROJECT_NAME>/<NN-slug>/<NN-slug>.mp4 — directly under AI_BlogSite/, not inside Videos/. Every 10 videos becomes a new folder named after the project. Persisted in memory.
  • Pexels backdrops as the default for hook scenes. The Pexels MCP went live at the tail end of Vol 2. Vol 3 will use it from V21 onward — every Scene 1 gets a stylized, dimmed, blurred laptop / desk / hand B-roll layer behind the hero text. Tested visually with V19; downloaded to AI_BlogSite/Assets/pexels/.
  • White-space budgets at draft. Every scene's vertical budget gets used at first draft. No more "ship and iterate when feedback hits 'too much empty space.'" Every scene has at least: hero, content row, payoff block, stats band.
  • Lock the C-variant aesthetic family in advance. Vol 2's C variants drifted (paper worksheet, dark digital lesson page, illustrated blueprint). Vol 3 starts with one C-variant identity — "illustrated dark lesson page" — and doesn't deviate.
  • Whisper-diff the VO against the brief before locking. Once the 5 VO files are generated, transcribe each, diff against the written brief, fix discrepancies before any visual work starts.
  • Stage the publish window. Vol 2's full 30 renders will not ship in 30 consecutive days. The cadence rules from Vol 1's audit still apply: TikTok daily, Instagram 3–4×/wk, YouTube every 2–3 days. That's a ~5-week publish window for V11–V20. Vol 3 will be built on top while Vol 2 is rolling out.
  • Instagram bio rewritten before V11 posts. Same flag from Vol 1 — 23 profile visits, 2 follows. Bio change is locked: name the niche, name the cadence, bake a follow CTA into the last 2s of every Reel.
  • Cross-link from V01 and V07. Vol 1's two breakouts (Google Drive AI, Copilot Cowork) still pull residual impressions. Every V11+ Short gets a pinned comment on those two driving traffic into the new series.
// The One Thing To Remember

The First 10 (Keyvello) shipped 4,407 views with 30+ errors — a production loop with no review step. The prompting series fixed the loop: Vol 1 proved the craft (3 followers, the hook thesis), and Vol 2 is now proving it scales — all ten lessons live, the channel's most-watched video ever, its first subscriber, and ~11.6K views climbing across platforms. The point of Vol 2 was to control the variables. The early numbers say the one that mattered most — a sharp, specific hook — is already paying off.
